### 🔴 Patrón sistémico crítico
El problema más acuciante y transversal en Life OS es la **falta de renderizado condicional y la duplicidad de contenido**, lo que se manifiesta en múltiples módulos donde el `_fold` y el `_scroll` son idénticos, o donde las pestañas internas no cambian el contenido. Esto no solo genera una experiencia de usuario rota y confusa, sino que también indica un problema fundamental en la arquitectura de la SPA (`main.js`) para gestionar estados y vistas dinámicas. La app no está respondiendo a las interacciones básicas del usuario ni a los cambios de estado, lo que impacta directamente en la usabilidad, la retención y la percepción de calidad.

Este fallo sistémico se agrava con la **ausencia de estados vacíos con personalidad y skeleton loaders**, lo que deja al usuario frente a pantallas estáticas o con datos incompletos, sin una guía clara sobre qué hacer o qué esperar. La falta de feedback visual y la inconsistencia en la aplicación de los principios de diseño de 2026 (glassmorphism, micro-animaciones, jerarquía tipográfica) en los estados críticos (onboarding, paywall, blackout) sugieren que la implementación actual es una maqueta estática en lugar de una aplicación dinámica y reactiva.

### 🐛 Bug de layout — diagnóstico

🔴 SENIOR QA ENGINEER: El screenshot `00-landing-fold` muestra un bug de layout crítico. Hay un espacio vacío significativo en la parte superior del hero, y los CTA principales ("Empezar gratis" y "Ya tengo cuenta") están cortados, no son visibles above the fold. Esto es una falla de conversión crítica.

🟠 SENIOR FRONTEND DEV: La causa raíz es la combinación de `min-height:calc(100dvh-53px)` en `.lp-hero` y la forma en que `flex-direction:column` en `#landing-page` distribuye el espacio. El `min-height` del hero está forzando un tamaño mínimo que, al estar centrado con `align-items:center` (asumo que es el estilo aplicado al hero o a su contenedor interno), empuja el contenido hacia abajo. El `flex:1` en `.lp-scroll` está haciendo su trabajo, pero el contenido dentro de `.lp-hero` no está optimizado para el espacio disponible. El `min-height` original intentaba que el hero ocupara todo el viewport menos el nav, pero al centrar el contenido, deja el espacio superior vacío.

**Fix CSS específico:**
Necesitamos que el contenido del hero se alinee al inicio del `.lp-hero` y que el `.lp-hero` no tenga un `min-height` tan restrictivo que empuje el contenido. En su lugar, el `.lp-scroll` debe gestionar el scroll y el `.lp-hero` debe tener un `padding-top` adecuado para el nav.

```css
/* main.js o archivo CSS relevante */

/* Eliminar el min-height problemático del hero */
.lp-hero {
  /* min-height: calc(100dvh - 53px); /* ELIMINAR ESTA LÍNEA */
  display: flex; /* Asegurar que el hero es un flex container */
  flex-direction: column; /* Para apilar el contenido verticalmente */
  justify-content: flex-start; /* Alinear contenido al inicio */
  align-items: center; /* Centrar horizontalmente */
  padding-top: 100px; /* Ajustar este valor para dar espacio al nav y un poco más */
  padding-bottom: 50px; /* Espacio inferior para que no se vea pegado al siguiente módulo */
  box-sizing: border-box; /* Asegurar que padding no cause overflow */
}

/* Asegurar que el contenedor principal maneje el espacio correctamente */
#landing-page {
  display: flex;
  flex-direction: column;
  height: 100dvh; /* Asegurar que ocupe todo el viewport */
}

.lp-scroll {
  flex: 1;
  overflow-y: auto;
  -webkit-overflow-scrolling: touch; /* Mejorar el scroll en iOS */
}

/* Asegurar que el nav no interfiera con el flujo */
.lp-nav {
  position: sticky; /* O fixed, dependiendo del comportamiento deseado */
  top: 0;
  z-index: 1000;
  height: 53px; /* Mantener la altura definida */
  width: 100%;
}
```
Con estos cambios, el contenido del hero debería empezar más arriba, haciendo visibles los CTA principales.

**Descripción de implementación**:
1.  **Identificar el elemento del banner**: Localizar el `div` o componente que renderiza el banner "MODO PRUEBA" (probablemente `#promo-banner` o similar).
2.  **Ajustar el CSS del banner**: Asegurar que el banner tenga `position: fixed; top: 0; left: 0; right: 0; z-index: 1000;` (o un `z-index` apropiadamente alto).
3.  **Compensar el contenido principal**:
    *   **Opción A (preferida)**: Aplicar un `padding-top` dinámico al `body` o al contenedor principal de la aplicación (`#app-container`) que sea igual a la altura del banner. Esto se puede hacer con JavaScript al cargar la app, midiendo la altura del banner y aplicando el estilo.
        ```javascript
        // main.js (o un script de inicialización)
        document.addEventListener('DOMContentLoaded', () => {
            const promoBanner = document.getElementById('promo-banner'); // Asumiendo este ID
            if (promoBanner) {
                const bannerHeight = promoBanner.offsetHeight;
                document.body.style.paddingTop = `${bannerHeight}px`;
                // Para iOS safe area, considerar `env(safe-area-inset-top)` en el cálculo
                // o aplicar un `padding-top` adicional si el banner no lo cubre.
            }
        });
        ```
    *   **Opción B**: Si el banner es parte de un componente de navegación superior, asegurar que el resto del contenido de la app se renderice *debajo* de este componente, quizás usando `display: flex; flex-direction: column;` en el contenedor principal y dando al banner una altura fija o `flex-shrink: 0`.
4.  **Considerar Safe Area en iOS**: Para iOS, es crucial usar `safe-area-inset-top` en el CSS para asegurar que el banner no invada el área del notch. Esto podría implicar un `padding-top` adicional o un `min-height` para el banner que incluya esta variable.
    ```css
    /* En el CSS del banner */
    .promo-banner {
        padding-top: env(safe-area-inset-top);
        /* ... otros estilos ... */
    }
    /* Y luego el padding-top del body se ajustaría a la altura total del banner incluyendo el safe-area */
    ```
Esta solución garantizará que el contenido de la app sea siempre visible y que la experiencia de usuario no se vea comprometida por elementos superpuestos.
